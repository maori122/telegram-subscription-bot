# 📖 Руководство по использованию

## Для пользователей

### 1. Регистрация

Отправьте боту `/start`

Вы увидите:
```
Привет! Я бот, который выдает рекламные ссылки.

Нажми ✅ чтобы зарегистрироваться

[✅ Зарегистрироваться]
```

Нажмите кнопку.

### 2. Получение ссылки

Когда админ разошлет ссылку, вы получите:
```
🔗 Новая рекламная ссылка:

https://t.me/example_channel

После подписки отправьте скриншот в этот чат.
```

### 3. Подписка

1. Перейдите по ссылке
2. Подпишитесь на канал/группу
3. Сделайте скриншот

### 4. Отправка скриншота

Просто отправьте фото скриншота в бота.

Вы получите:
```
Зафиксировано 1 подписок
```

При следующей подписке:
```
Зафиксировано 2 подписок
```

И так далее!

## Для администраторов

### 1. Авторизация

Отправьте:
```
/admin Yp~?L*W<a"v/(~mFm&[&
```

Сообщение автоматически удалится, и вы увидите:
```
✅ Вы авторизованы как администратор!

🤖 Панель администратора

[👥 Список пользователей]
[🔗 Разослать ссылку   ]
[💬 Настроить ответ    ]
```

### 2. Просмотр пользователей

Нажмите **👥 Список пользователей**

Вы увидите список:
```
👤 Выберите пользователя для управления:

[Иван (@ivan123) - 5 подписок    ]
[Мария (@maria456) - 3 подписки  ]
[Петр (@petr789) - 0 подписок    ]

[🔙 Назад]
```

### 3. Управление подписками

Выберите пользователя, появится:
```
👤 Иван (@ivan123)

📊 Подписок: 5
📅 Регистрация: 26.02.2026

[➕ +1 подписка] [➖ -1 подписка]
[     🔄 Обнулить              ]
[     🔙 К списку              ]
```

**Действия:**
- **➕ +1** - увеличить на 1
- **➖ -1** - уменьшить на 1
- **🔄 Обнулить** - сбросить в 0

### 4. Рассылка ссылки

Нажмите **🔗 Разослать ссылку**

Бот попросит:
```
🔗 Отправьте ссылку, которую нужно разослать всем пользователям:

[❌ Отмена]
```

Отправьте ссылку, например:
```
https://t.me/my_channel
```

Бот разошлет всем и покажет статистику:
```
✅ Ссылка разослана!

Успешно: 15
Ошибок: 0
```

### 5. Настройка ответа

Нажмите **💬 Настроить ответ**

Бот попросит:
```
💬 Напишите текст ответа, который будет отправляться 
после получения скриншота:

Используйте {count} для вставки количества подписок.
Например: "Зафиксировано {count} подписок"

[❌ Отмена]
```

Отправьте свой текст:
```
✅ Отлично! Зафиксировано {count} подписок! 
Спасибо за участие! 🎉
```

Теперь пользователи будут получать этот текст.

### 6. Уведомления

Когда пользователь отправляет скриншот, вы получите:
```
📸 Новый скриншот от Иван (@ivan123)
Всего подписок: 6
```

## Примеры использования

### Пример 1: Простая рассылка

1. Админ: Нажимает "🔗 Разослать ссылку"
2. Админ: Отправляет `https://t.me/channel`
3. Все пользователи получают ссылку
4. Пользователи подписываются и отправляют скриншоты
5. Админ видит уведомления о каждом скриншоте

### Пример 2: Корректировка подписок

Пользователь забыл отправить скриншот, но подписался:

1. Админ: "👥 Список пользователей"
2. Админ: Выбирает пользователя
3. Админ: Нажимает "➕ +1 подписка"
4. Счетчик увеличивается

### Пример 3: Сброс статистики

Начало нового месяца, нужно обнулить:

1. Админ: "👥 Список пользователей"
2. Админ: Выбирает пользователя
3. Админ: Нажимает "🔄 Обнулить"
4. Счетчик = 0

### Пример 4: Кастомный ответ

Хотите мотивировать пользователей:

1. Админ: "💬 Настроить ответ"
2. Админ: Отправляет:
```
🎉 Супер! Это уже {count} подписок!

Продолжайте в том же духе! 
Скоро будет приз! 🏆
```
3. Теперь пользователи получают мотивирующее сообщение

## Частые вопросы

**Q: Что если я отправлю не скриншот?**
A: Бот обрабатывает только фото. Текст игнорируется.

**Q: Можно ли отправить несколько скриншотов сразу?**
A: Да, каждое фото увеличит счетчик на 1.

**Q: Админ видит мои скриншоты?**
A: Админ получает только уведомление о факте отправки, не сам скриншот.

**Q: Можно ли удалить подписку?**
A: Админ может уменьшить счетчик кнопкой "➖ -1".

**Q: Сколько админов может быть?**
A: Неограниченно. Каждый, кто знает пароль, может авторизоваться.

## Технические детали

**Хранение данных:**
- Пользователи: имя, username, количество подписок, дата регистрации
- Админы: список авторизованных
- Настройки: текст ответа

**Автоматизация:**
- Подсчет подписок при получении фото
- Уведомления админам
- Подстановка {count} в текст ответа

**Безопасность:**
- Пароли не хранятся в коде
- Сообщения с паролями удаляются
- Только админы видят статистику
